## Big Picture

Fast logbook PWA is a deliberately small, privacy-first, offline-first work-time logger: one textarea, nine shortcuts, one export button. The 2026 rewrite (vanilla JS → React 19 + TypeScript + Vite + IndexedDB) preserved the log format and UX contract while replacing every infrastructure layer. `docs/ADR.md` records the decisions, including the removal of GA4 analytics entirely (ADR-005).

## The Rewrite Fault Line

The single most important thing to know when working here: **documentation freshness splits along `docs/` vs `.claude/`**.

| Layer | Status |
| --- | --- |
| `docs/design.md`, `docs/ADR.md`, `docs/spec/*` | Rewritten for React era — trustworthy (Factual, verified against `src/`) |
| `.claude/CLAUDE.md`, `.claude/rules/code-style.md`, `.claude/rules/security.md` | Describe the deleted `js/` + hand-written `sw.js` world — **do not follow their file paths, module rules, or bug lists** |
| `tests/e2e/screenshot.spec.ts` | Half-migrated (`/config.html` stale URL) |

## Non-obvious Behaviors

- **Logical day ≠ calendar day**: the roll-over time (default 05:00) shifts day boundaries; `getDateBoundaries()` builds a [date+rollOver, +24h) window. Entries before 05:00 belong to the previous logical day.
- **Two-stage persistence**: keystrokes → localStorage buffer (300ms debounce) → IndexedDB merge on tab-hide/date-change/load. The green/red navbar dot reflects buffer state, not IndexedDB state.
- **Popup-blocker choreography**: the log viewer window is opened *synchronously before any `await`* in `handleViewLog()`; the download flow uses a Blob URL + CustomEvent + programmatic `<a>` click. Both are deliberate workarounds — keep the ordering.
- **IME handling**: Enter-key logic checks `isComposing` and legacy `keyCode === 229` — hard-won Japanese-input fix, preserved from the old codebase; don't "simplify" it away.
- **Version display**: the UI version comes from `manifest.json` fetched at runtime (generated by vite-plugin-pwa from `vite.config.js`), not from `package.json`.
