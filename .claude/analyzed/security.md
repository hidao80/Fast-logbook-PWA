## Threat Model

Client-only PWA, no backend, no authentication, no secrets, no user data transmission (GA4 analytics fully removed — ADR-005). The user is both the only data source and the only data consumer. Attack surface: served static assets, the exported standalone HTML artifact, third-party supply chain. (Factual)

### A03 detail — XSS paths

1. **Log viewer** (`window.open` + `document.write` in [src/App.tsx](../../src/App.tsx#L398)): content is `generateFormattedLog()` output; user-controlled category/detail/plaintext go through `escapeHtml()` ([src/lib/download.ts](../../src/lib/download.ts)); table cells and `<pre><code>` sections are escaped. Verified escaping on all user-content interpolation points. Residual: self-XSS in the user's own file only. (Factual)
2. **`dangerouslySetInnerHTML`** in `App.tsx` (help/notice modals): input is **static translation JSON** (`src/i18n/locales/*.json`), never user data. Biome's `noDangerouslySetInnerHtml` is deliberately disabled — safe *only while this invariant holds*. Guard rule: never pass user/storage-derived strings there.
3. **i18next `interpolation.escapeValue: false`**: currently no `t()` call interpolates user variables, so no exposure (Factual at this commit); becomes a risk if interpolated translations are added later.

## Additional Categories

- **Secrets**: none in repo (no API keys, tokens, .env). (Factual — nothing to leak by design)
- **Privacy / data exfiltration**: zero external requests from the app shell at runtime except same-origin `manifest.json`; exported HTML loads CDN assets *when the user opens that file*. No PII ever leaves the browser.
- **Input validation**: `getRoundingUnit()` whitelists values; date/roll-over come from native `type=date`/`type=time` inputs; log text is free-form by design and always escaped on output. QuotaExceededError handled on all writes.
- **Clickjacking / headers**: no `X-Frame-Options`/`frame-ancestors` (low relevance — no auth to steal, but trivially addable with the CSP work).
- **Service worker**: generated by Workbox (vite-plugin-pwa), same-origin precache only; no runtime caching of cross-origin content configured. (Factual from config)
