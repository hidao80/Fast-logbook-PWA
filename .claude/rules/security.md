# Security Rules

Security policy for this project. Assumes a client-only PWA with no backend.

## Core Principles

- **No backend** — all data is stored exclusively in the browser (`localStorage`)
-  No log content or personally identifiable information is sent to any server or external API
- Never embed secrets (API keys, tokens) in code

## Escaping HTML Output

Always use `escapeHtml()` (from `js/lib/utils.js`) when inserting user input into HTML.

```js
// OK
`<td>${escapeHtml(category)}</td>`

// NG — XSS risk
`<td>${category}</td>`
```

`escapeHtml()` escapes these 5 characters: `&`, `<`, `>`, `"`, `'`.
Generated HTML is used only as a downloaded file, but apply the same rule when inserting into `document.write()` or similar.

## Use of `document.write()`

The `view_formatted_log` handler in `js/main.js` uses `newTab.document.write(outputStr)`.
`outputStr` is the full HTML generated by `generateFormattedLog()`. Before calling `document.write()`, verify that all user-controlled content is escaped via `escapeHtml()`.
If you add a new use of `document.write()`, verify that all inserted content is fully escaped.

## localStorage

- Use the constants in `utils.js` for key names (`LOG_DATA_KEY`, `ROUNDING_UNIT_MINUTE_KEY`)
- `localStorage.getItem()` can return `null` or the string `'undefined'`; always check for existence before use
- `localStorage.setItem()` can throw `QuotaExceededError`; wrap write calls in try-catch

## CDN Resources

`sw.js` and `download.js` load resources from CDNs (jsDelivr, cdnjs).
CDN `<script>` and `<link>` tags in the HTML generated by `download.js` include SRI (`integrity` attribute).
When changing a CDN URL or version, always update the SRI hash as well.

```html
<!-- OK: integrity present -->
<script src='https://cdnjs.cloudflare.com/...' integrity='sha512-...' crossorigin='anonymous'></script>

<!-- NG: no integrity -->
<script src='https://cdnjs.cloudflare.com/...'></script>
```

## GA4 Analytics

`js/lib/analytics.js` sends events to Google Analytics 4.
Only app interaction events are sent — never log content or personally identifiable information.
The `gtag` call is guarded against `undefined` (see `initAnalytics()`).

## Service Worker Cache Pollution

There is a known bug in the `sw.js` fetch handler: it writes to the hardcoded key `'my-cache'` instead of `CACHE_NAME`.
When fixing this bug, use the `CACHE_NAME` variable. Writing to the wrong cache key causes stale cache entries to persist across version upgrades.

## Development Server

`npm run dev` does not use HTTPS. Modern browsers treat `localhost` as a secure context, so Service Worker registration and testing work over HTTP on `localhost`.
HTTPS is required only when testing on non-localhost origins (e.g. a staging server or a device on the local network). In those cases, use a local HTTPS proxy or the production environment.
